Detailed Description

Defines methods for managing user roles within an application.

This service provides both synchronous and asynchronous methods for adding and removing roles from users. It supports operations using either a user object or an email identifier.

Template Parameters
TUserThe type of user, which must inherit from ApplicationUser.

Adds a specified role to a user.

This method checks if the specified role exists and if the user is already present in the system before attempting to add the role. It also ensures that changes are saved successfully.

Parameters
userThe user to whom the role will be added. Cannot be null.

Returns
An AccessResult indicating the success or failure of the operation. Returns AccessResult.Success if the role is successfully added. Returns AccessResult.Failed with an appropriate error message if the role is invalid, the user does not exist, or if saving changes fails.

Asynchronously adds a specified role to a user identified by their email address.

This method checks if the specified role exists and if the user with the given email exists. If either does not exist, the operation fails with a corresponding error message.

Returns
An AccessResult indicating the success or failure of the operation. Returns AccessResult.Success if the role is successfully added; otherwise, returns AccessResult.Failed with an appropriate error message.
